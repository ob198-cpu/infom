System Development

現場の作業に合わせて、必要な機能から作ります。

Approach

最初から全部作らず、使う部分から始めます。

業務システムは、機能が多いほど良いとは限りません。 InfoMでは、最初に作る範囲、後で足す範囲、作らない方がよい範囲を分け、運用できるサイズで始めます。

Use Cases

対応しやすい業務

Excel管理のWeb化

複数ファイル、転記、最新版管理を減らし、入力と確認を一つの流れにします。

申請・承認管理

申請内容、確認者、ステータス、履歴を見えるようにします。

顧客・案件管理

顧客情報、対応履歴、次回予定、進捗をまとめて管理します。

相談・面談記録

記録の入力、権限、対応状況、次回確認を整理します。

ストレスチェック運用管理

受検状況、本人通知、簡易報告など、制度運用に関わる作業を支援します。

研修・講義管理

受講者、日程、資料、アンケート、振り返りを管理しやすくします。

Build Scope

開発で整える範囲

領域 内容 目的
入力フォーム、必須項目、選択肢、入力補助転記や入力漏れを減らす
一覧検索、絞り込み、並び替え、ステータス表示今見るべき情報を探しやすくする
詳細対応履歴、添付、メモ、次回予定担当者だけに情報が残らないようにする
集計件数、進捗、月次集計、CSV出力報告や振り返りに使える形にする
権限閲覧範囲、編集範囲、管理者画面見せてよい情報を分ける

Process

開発の流れ

01現状確認

今の作業、使っている表、困っている場面を確認します。

02範囲整理

最初に必要な機能と、後から足せる機能を分けます。

03試作

画面や入力の流れを作り、実際の業務に合わせて調整します。

04運用開始

使い方、権限、修正方法、保守の進め方を整理します。

Cost

費用は、作る画面数だけでは決まりません。

費用に影響するもの

利用者数、権限、データ量、帳票、外部連携、公開範囲、保守方法。

小さく始める目安

業務整理は33,000円から、小規模ツールは165,000円から、Webシステムは330,000円から。

見積前に確認すること

いま何に時間がかかっているか、誰が使うか、どの情報を見せてよいか。

今の業務を、どこからシステム化するか相談する

要件が固まっていなくても、現状のExcelや手順から整理できます。
